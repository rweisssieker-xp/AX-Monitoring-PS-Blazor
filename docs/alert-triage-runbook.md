## Alert Triage Runbook

1. Eingang
   - Kanal (E-Mail/Teams), Dedupe-Key, Zeitstempel prüfen
2. Kontext
   - Betroffene Klasse/AOS/DB, zugehörige Metriken öffnen
3. Maßnahmen
   - Wartungsfenster aktiv? ggf. schließen
   - Stummschalten bei bekannten False Positives
4. Eskalation
   - On-Call/DBA/Ops gemäß RACI
5. Abschluss
   - Alert schließen/kommentieren, Lessons learned sammeln
